## Choose a path

| Path  | Web    | API                       | When                                            |
| ----- | ------ | ------------------------- | ----------------------------------------------- |
| **A** | Vercel | Vercel (`vercel-handler`) | Minimal hosts; accept serverless limits         |
| **B** | Vercel | Render Docker             | KitsuneKode default; blueprint in `render.yaml` |
| **C** | Vercel | Railway Docker            | `railway.toml` deploy                           |

**Postgres and Redis are always URL-based** on the API host. Do not use Render Postgres/Key Value for this template—use [Neon](https://neon.tech) and [Upstash](https://upstash.com) (or any compatible URL provider).

## Rollout order

1. Provision Postgres + Redis (or `ENABLE_REDIS=false` for API-only)
2. Deploy API; set `DATABASE_URL`, auth URLs, secrets
3. `bun run db:migrate` with production `DATABASE_URL`
4. `curl https://<api>/health`
5. Deploy web; set `NEXT_PUBLIC_API_URL`
6. Smoke: sign-in, one tRPC call, check browser CORS
